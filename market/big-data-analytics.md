# 大数据承载世界

目前，大数据已经无处不在。92%的企业和IT执行部门都看到了大数据对于他们业务的重要性（来源： Wikibon, 2014）。使用案例跨度非常大，有：目标广告、电子商务推荐引擎、销售卖点事务分析、物联网数据分析、IT系统监控、客户流失管理、贸易监测、风险建模、威胁分析以及搜索分析。当你拥有多个数据来源（web、无线等）的场景时，加上企业中的部门，以及每个场景用到的集群类型（QA、Test、生产等。）具体排列出来是这样子的：

```集群数量``` = ```场景数量``` x ```数据来源``` x ```部门数``` x ```多集群／部门```

要满足这些需求的话，IT／运维部门团队该创建大量的小的集群，那么就陷入了如下的两难境地：

* 手动的对这些大数据集群进行部署、管理、监控、升级和备份。
* 在手动做上述工作的时候，遭到了数据科学家的督促和抱怨，因为他们必须等待初始的部署和变更管理。
* 当应用要生产上线时管理性能困难，且需要动态扩展。
* 非常低的利用率，尽管坐拥昂贵的软硬件设施。
* 无法共享数据等于又建立了数据竖井，结果就是增加更多的人手去传输数据。

有些IT/运维部门或许会尝试创建几个较大的集群来解决上述问题，但是这根本就不是解决问题的最终办法。较大规模集群带来的问题有，性能扩展困难、多租户的解决、数据收集、和其它大数据工具数据集成困难、公司范围内 的软件和政策标准、备份和灾难恢复、更低的资源利用率、以及这些大型集群的声明周期管理。

使用 Mirantis OpenStack 自助大数据集群平台，是解决上述方法的最佳之道，Sahara 项目，隶属于 Mirantis OpenStack 的一部分，让用户能够在OpenStack 的云平台上一键部署 Hadoop或Spark集群，而且和OpenStack Swift以及相兼容的存储结合起来，可以解决数据共享、备份、灾难恢复、以及最小化ETL（提取、转化、加载）代码等解决方案。

## 面向数据科学家：集群一键部署

当你的业务需要分析出答案时，你恨不得数据科学家们立刻就给出来！而数据科学家们希望Hadoop或Spark集群立刻就准备就绪，你肯定不想提交一工单，然后登上几周或几个月，然后集群才可用了。更何况你还需要合适的软件发行版（例如，是Cloudera的hadoop版本，还是 Hortonworks 数据平台，还是Apache Apark）来安装以及进行特定的配置，Mirantis OpenStack 以及其组件 Sahara ，可以通过 Horizon 或 API 来进行自助式的一键部署Hadoop或Spark集群。在开发阶段根据你的负载自动匹配合适的集群；一键部署不同大小的集群。而且当你的应用需要生产上线时，你可以轻松扩展集群以达到增长的性能需求。另外，你还可以随时通过回收集群资源。所有的这一切都在Mirantis OpenStack中解决。

## IT/运维人员：大数据集群生命周期

尽管有50%的 IT／运维人员相信是他们提供了大数据价值，但是只有18%的用户同意这一说法（来源： Wikibon, 2014），导致如此说法不一致的主要原因是手动的部署和变更管理，而这也是用户和IT/运维人员的分歧所在。充分利用Mirantis OpenStack 和 Sahara的能力，你可以简化部署，自助式扩展Hadoop集群，使用节点模版，你可以控制基于规则治理的集群，而这一切都可以通过在集群内部的节点定义VM的规模和角色来完成。你也可以通过图形界面或API来完成基于角色的节点分组和组内部节点的扩展——让你和你的用户均能够完全的控制集群。Sahara管理着大数据集群的整个生命周期，从最初的部署，到拆除、优化资源利用率。OpenStack 也解决的让你头痛的多租户问题——基于项目分离的每个集群都可以利用OpenStack的多租户。另外，你可以使用HDFS或者是Swift兼容的存储，Swift 简化了数据的共享，需要移动，传输和复制的数据。使用弹性的对象存储，可以跨地理位置，并可实现备份的功能。

总结一句话：Mirantis OpenStack 帮助你从手动创建Hadoop和Spark集群中解放出来。

## 企业领导者：改进的敏捷

作为企业领导者，你打算使用大数据来提高自己的竞争优势。那么可能最可能将这一过程拖慢的就是你的基础设施。

Mirantis OpenStack 通过交付 Sahara项目，为大数据科学家和开发者提供如下功能：

* 即时访问完全配置好的 Hadoop 或 Spark 集群。
* 根据性能实际使用情况，随时扩展。
* 能够按需来随时构建和拆除集群。
* 能够通过使用和 Swift 在后端兼容的存储来分享数据。
* 显著减少应用程序和数据仓库之间的代码同步。

同时，作为运维人员，则能够感受到如下好处：

* 通过使用自助式的部署 Hadoop和Spark集群，消除了手动工作从而降低操作费用(OPEX)
* 提高设备的资源利用率和削减资本支出(资本支出)
* 能够从 AWS Elastic Map Reduce (EMR) 负载中迁移出来到OpenStack，消除 Shadow IT，改善遵从性和降低成本。
